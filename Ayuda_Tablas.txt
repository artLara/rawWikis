<p>En Wikipedia, una <b>tabla</b> es una forma de representar mucha información de manera esquematizada, ordenada y compacta. Aunque Wikipedia acepta el código <a href="/wiki/HTML" title="HTML">HTML</a> para la creación de tablas como en cualquier <a href="/wiki/P%C3%A1gina_web" title="Página web">página web</a>, posee también un código propio, cuya <a href="/wiki/Sintaxis" title="Sintaxis">sintaxis</a> basada en barras verticales resulta bastante más limpia y compacta.
</p>
<p>Los parámetros de tabla y de celdas son los mismos como en HTML, ver <a class="external autonumber" href="http://www.w3.org/TR/html401/struct/tables.html#edef-TABLE" rel="nofollow">[1]</a> (página en inglés) y <a class="extiw" href="https://en.wikipedia.org/wiki/HTML_element#Tables" title="w:HTML element">HTML element#Tables</a> (wiki en inglés). Sin embargo los elementos <tt>thead</tt>, <tt>tbody</tt>, <tt>tfoot</tt>, <tt>colgroup</tt> y <tt>col</tt> actualmente no son <a class="mw-redirect" href="/wiki/Html#Permitted_HTML" title="Html">soportados en MediaWiki</a>.
</p>
<p>Una tabla puede ser útil aún si ninguna de sus celdas tiene contenido, notablemente porque con el uso de colores de fondo de celdas la tabla puede ser un diagrama; véase por ejemplo <a href="/wiki/Ajedrez#El_tablero_de_ajedrez" title="Ajedrez">Ajedrez#El tablero de ajedrez</a>. En algunos casos, una "imagen" en forma de tabla es mucho más conveniente para ser editada que un archivo de imagen propiamente dicho.
</p>
<p>Cada fila debe tener el mismo número de celdas que las otras filas, así la cantidad de columnas en la tabla continúa siendo consistente (a menos que haya celdas que abarquen varias columnas o filas, véase <tt>colspan</tt> y <tt>rowspan</tt> en ejemplo mezcla más abajo). Para celdas vacías, use el espacio “non-breaking” <tt>&amp;nbsp;</tt> como contenido para asegurar que las celdas sean mostradas.
</p>
<p>Como se expone a continuación, para simplificar el problema del código redundante necesario en muchas ocasiones para definir tablas por el procedimiento descrito anteriormente, se dispone de la opción <b><code>class="wikitable"</code></b>, que facilita <b>TABLAS PREDEFINIDAS</b> (que aquí se denominan de forma abreviada <i>Wikitablas</i>), dotadas de una apariencia estandarizada, y que son mucho más fácilmente configurables y con menos necesidad de código que las aquí denominadas a partir de ahora <i>Tablas normales</i> (es decir, las que no tienen declarado el parámetro "class="). 
</p>

<p>En Wikipedia, una <b>tabla</b> es una forma de representar mucha información de manera esquematizada, ordenada y compacta. Aunque Wikipedia acepta el código <a href="/wiki/HTML" title="HTML">HTML</a> para la creación de tablas como en cualquier <a href="/wiki/P%C3%A1gina_web" title="Página web">página web</a>, posee también un código propio, cuya <a href="/wiki/Sintaxis" title="Sintaxis">sintaxis</a> basada en barras verticales resulta bastante más limpia y compacta.
</p>
<p>Los parámetros de tabla y de celdas son los mismos como en HTML, ver <a class="external autonumber" href="http://www.w3.org/TR/html401/struct/tables.html#edef-TABLE" rel="nofollow">[1]</a> (página en inglés) y <a class="extiw" href="https://en.wikipedia.org/wiki/HTML_element#Tables" title="w:HTML element">HTML element#Tables</a> (wiki en inglés). Sin embargo los elementos <tt>thead</tt>, <tt>tbody</tt>, <tt>tfoot</tt>, <tt>colgroup</tt> y <tt>col</tt> actualmente no son <a class="mw-redirect" href="/wiki/Html#Permitted_HTML" title="Html">soportados en MediaWiki</a>.
</p>
<p>Una tabla puede ser útil aún si ninguna de sus celdas tiene contenido, notablemente porque con el uso de colores de fondo de celdas la tabla puede ser un diagrama; véase por ejemplo <a href="/wiki/Ajedrez#El_tablero_de_ajedrez" title="Ajedrez">Ajedrez#El tablero de ajedrez</a>. En algunos casos, una "imagen" en forma de tabla es mucho más conveniente para ser editada que un archivo de imagen propiamente dicho.
</p>
<p>Cada fila debe tener el mismo número de celdas que las otras filas, así la cantidad de columnas en la tabla continúa siendo consistente (a menos que haya celdas que abarquen varias columnas o filas, véase <tt>colspan</tt> y <tt>rowspan</tt> en ejemplo mezcla más abajo). Para celdas vacías, use el espacio “non-breaking” <tt>&amp;nbsp;</tt> como contenido para asegurar que las celdas sean mostradas.
</p>

<p>Si quieres forzar el ancho de columna de acuerdo a tus necesidades en vez de aceptar el ancho del mayor elemento en una celda de una columna, sigue este ejemplo. Nótese que el espacio alrededor del texto está impuesto. La instrucción utilizada es <code>width=<i>n</i>px</code> (que fija el ancho en <i>n</i>px; también admite el formato con el valor entrecomillado en la forma <code>width="<i>n</i>"</code>); pudiendo usarse también el formato <code>width=<i>p</i>%</code> (donde <i>p</i> es el porcentaje del ancho de la tabla que se desea utilizar). Esta orden (en sus dos formatos) se pueden aplicar tanto a columnas individuales como a tablas completas (por ejemplo, fijando el ancho total de la tabla en su encabezamiento en <i>n</i>px, y repartiendo las columnas en <i>p</i>%).
